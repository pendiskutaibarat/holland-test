01 · PRD

Product Requirements Document

Holland Test adalah platform asesmen minat dan kecenderungan karir berbasis teori Holland RIASEC. Produk ini memindahkan asesmen dari pengalaman tunggal di browser menjadi platform berbasis sesi, tempat admin atau guru membuat link tes, siswa mengisi asesmen, dan hasil tersimpan di PostgreSQL untuk ditinjau, dibagikan, dan diekspor.

Ringkasan Produk

Aplikasi menyediakan tes Holland RIASEC untuk dua kebutuhan utama: pemetaan peminatan SMA/MA dan eksplorasi karir/program studi. Siswa mengakses tes dari link sesi, mengisi identitas awal, memilih atau mengikuti mode yang ditentukan guru, menjawab pertanyaan RIASEC, lalu langsung menerima hasil. Guru melihat agregasi hasil di dashboard, membuka detail siswa, mengunduh rekap Excel, dan dapat membagikan akses sesi ke guru lain melalui fitur kolaborator.

6 dimensi RIASEC: Realistic, Investigative, Artistic, Social, Enterprising, Conventional.
2 mode hasil utama: Peminatan SMA/MA dan Karir & Program Studi.
8 langkah wizard pada implementasi saat ini: pemilihan mode atau mode terkunci, enam bagian RIASEC, dan hasil.
5 model data inti: User, Session, TestResult, ResultAnswer, SessionCollaborator.

Masalah yang Diselesaikan

Sebelum rework platform, asesmen berjalan sebagai aplikasi client-side dan penyimpanan hasil diarahkan ke Google Sheets/Apps Script. Pola itu membuat guru sulit mengelola banyak sesi, memisahkan hasil per kelas/sekolah, mengontrol akses tes, atau melihat riwayat siswa dari satu dashboard. Sistem saat ini menyelesaikan masalah tersebut dengan otentikasi, sesi tes unik, penyimpanan database, dan tampilan detail hasil.

Tujuan produk bukan menggantikan konseling, melainkan menyediakan bahan awal yang rapi, terukur, dan mudah dibahas oleh guru BK bersama siswa.

Tujuan

  • Guru dapat membuat sesi tes dengan nama sesi, nama sekolah/madrasah, mode tes, dan deskripsi opsional.
  • Siswa dapat mengerjakan tes melalui link unik tanpa akun siswa.
  • Hasil siswa tersimpan di PostgreSQL dengan skor RIASEC, mode, persentase peminatan bila relevan, dan jawaban pertanyaan.
  • Admin dapat mengelola akun guru melalui status PENDING, ACTIVE, dan REJECTED.
  • Guru atau admin dapat melihat ringkasan hasil, detail hasil, kolaborator sesi, dan ekspor Excel.

Non-Tujuan

  • Aplikasi tidak menyediakan akun siswa atau autentikasi siswa.
  • Aplikasi tidak menggunakan SSO, LMS sekolah, atau integrasi email.
  • Aplikasi tidak menyediakan realtime dashboard; guru melihat hasil setelah refresh atau navigasi ulang.
  • Aplikasi bukan alat diagnosis psikologis final; hasilnya eksploratif dan perlu pendampingan guru BK/konselor.

02 · Scope

Ruang Lingkup Produk dan Pengguna

Produk memiliki tiga kelompok pengguna: admin, guru, dan siswa. Admin memegang kontrol akun guru; guru menjalankan asesmen; siswa hanya memakai link sesi untuk mengerjakan tes.

Persona Utama

PersonaKebutuhanFitur Terkait
AdminMengelola guru, menyetujui/menolak registrasi, dan melihat sesi.Login, dashboard tab Guru, approve/reject user.
Guru BK atau wali kelasMembuat sesi, membagikan link, memantau hasil, dan mengekspor rekap.Dashboard sesi, detail sesi, copy link, Excel export.
SiswaMengisi asesmen dengan instruksi sederhana dan menerima hasil langsung.Halaman /test/[code], wizard RIASEC, halaman hasil.
Guru kolaboratorMengakses sesi yang dibagikan tanpa menjadi pemilik sesi.SessionCollaborator, akses OWNED/SHARED.

Ruang Lingkup MVP Saat Ini

Scope aktif meliputi autentikasi email/password, registrasi guru dengan persetujuan admin, pembuatan sesi tes, halaman tes publik berdasarkan kode sesi, penyimpanan hasil ke database, tampilan detail hasil per siswa, ringkasan sesi, kolaborasi antar guru, dan ekspor Excel. PDF hasil siswa di implementasi saat ini memakai mekanisme cetak browser dari halaman hasil; library html2pdf.js masih tercatat sebagai dependency dan utilitas pendukung, tetapi tombol utama yang terlihat pada komponen hasil adalah Cetak Hasil.

Aturan Produk Penting

  • Sesi memiliki mode bebas, peminatan, atau karir. Mode bebas membuat siswa memilih mode; mode lain mengunci pilihan.
  • Sesi hanya dapat dikerjakan saat is_active bernilai true.
  • Guru berstatus PENDING atau REJECTED tidak boleh masuk ke dashboard.
  • Admin memiliki akses pengelolaan guru dan dapat melihat sesi; guru biasa hanya melihat sesi miliknya dan sesi yang dibagikan.
  • Hasil duplikat dicegah berdasarkan kombinasi sesi, nama siswa, dan field student_class.
Catatan Implementasi
Pada flow siswa saat ini, form awal meminta nama lengkap dan tanggal lahir. Nilai tanggal lahir diteruskan ke prop bernama studentClass dan ikut masuk ke payload sebagai student_class sekaligus birth_date. Dokumen ini mencatat perilaku aktual tersebut sebagai risiko data yang perlu dirapikan bila aplikasi membutuhkan kelas siswa sebagai field terpisah.
03 · Functional Flow

Alur Produk dan Kebutuhan Fungsional

Aplikasi dibangun sebagai rangkaian flow yang jelas: admin/guru login, guru membuat sesi, siswa mengerjakan tes, hasil disimpan, lalu guru meninjau dan mengekspor data.

Flow Admin dan Guru

Registrasi Guru Persetujuan Admin Login Dashboard Buat Sesi Bagikan Link Lihat Hasil
  • Guru baru mendaftar melalui /admin/register dan dibuat sebagai role TEACHER dengan status PENDING.
  • Admin membuka dashboard tab Guru, lalu menyetujui atau menolak akun melalui endpoint approve/reject.
  • Guru aktif login di /admin/login; token JWT disimpan pada cookie httpOnly bernama auth-token.
  • Guru membuat sesi dengan nama sesi, nama sekolah/madrasah, mode, dan deskripsi.
  • Sesi tampil sebagai kartu dengan jumlah hasil, status aktif, owner, dan tindakan melihat detail.

Flow Siswa

Buka /test/[code] Validasi Sesi Isi Identitas Pilih atau Ikuti Mode Jawab RIASEC Simpan Hasil Lihat Hasil
  • Halaman server /test/[code] mengambil metadata sesi melalui kode sesi.
  • Jika sesi tidak aktif, siswa melihat pesan sesi ditutup.
  • Jika sesi aktif, siswa mengisi nama lengkap dan tanggal lahir.
  • Wizard menampilkan pemilihan mode bila mode sesi bebas; jika mode dikunci, wizard langsung menampilkan mode yang ditentukan.
  • Pada tahap hasil, aplikasi mengirim POST /api/results satu kali dan menampilkan status sukses, duplikat, error, atau sedang mengirim.

Flow Detail Sesi

Halaman detail sesi menyajikan metadata sesi, link tes, daftar kolaborator, ringkasan jumlah hasil, rata-rata skor RIASEC, rata-rata persentase peminatan, daftar hasil siswa, dan tombol unduh Excel. Admin dapat menambah atau menghapus kolaborator sesi; guru kolaborator melihat akses sebagai SHARED.

Kebutuhan Fungsional

KodeKebutuhanStatus Implementasi
FR-01Autentikasi admin/guru dengan email dan password.Sudah ada via JWT, bcryptjs, cookie httpOnly.
FR-02Registrasi guru dengan approval admin.Sudah ada: TEACHER dibuat PENDING, admin approve/reject.
FR-03Pembuatan sesi tes dengan sekolah/madrasah wajib.Sudah ada di API dan UI dashboard.
FR-04Link sesi publik untuk siswa.Sudah ada di /test/[code].
FR-05Mode tes bebas atau terkunci.Sudah ada di WizardContainer dan API result validation.
FR-06Penyimpanan skor RIASEC dan jawaban pertanyaan.Sudah ada di TestResult dan ResultAnswer.
FR-07Ekspor hasil per sesi.Sudah ada sebagai ekspor Excel client-side.
FR-08Berbagi sesi ke guru lain.Sudah ada melalui SessionCollaborator.

Kebutuhan Non-Fungsional

  • Keamanan: password disimpan sebagai bcrypt hash, JWT wajib memakai JWT_SECRET, dan cookie bersifat httpOnly.
  • Reliabilitas data: database memakai foreign key cascade dan unique constraint untuk mencegah duplikasi hasil.
  • Performa: wizard tetap client-side agar interaksi pertanyaan cepat tanpa round-trip per langkah.
  • Keterbacaan hasil: halaman hasil langsung memuat narasi profil, skor, rekomendasi, dan tombol cetak.
  • Operasional: migrasi database dikelola dengan Prisma migration dan seed admin tersedia.
04 · RIASEC

Metodologi Holland RIASEC

Tes menggunakan enam domain Holland Occupational Themes. Setiap domain memiliki kumpulan pernyataan; skor dihitung dari jumlah pilihan siswa pada domain tersebut.

Enam Domain

DomainLabel IndonesiaMakna Produk
RealisticRealistisCenderung pada aktivitas praktis, alat, mesin, fisik, atau teknis.
InvestigativeInvestigatifCenderung pada analisis, riset, pemecahan masalah, dan penalaran.
ArtisticArtistikCenderung pada ekspresi kreatif, desain, bahasa, dan ide orisinal.
SocialSosialCenderung pada membantu, mengajar, mendampingi, dan berinteraksi.
EnterprisingWirausahaCenderung pada memimpin, memengaruhi, menjual, dan mengambil inisiatif.
ConventionalKonvensionalCenderung pada struktur, administrasi, data, prosedur, dan ketelitian.

Mode Peminatan SMA/MA

Mode peminatan mengubah skor enam domain menjadi persentase IPA, IPS, dan Bahasa & Budaya memakai bobot pada PEMINATAN_WEIGHTS. Jika semua skor kosong, sistem mengembalikan distribusi default 33.33%, 33.33%, dan 33.34% agar tampilan tidak rusak. Hasil menampilkan urutan peminatan, narasi bidang, mata pelajaran relevan, dan dua profil RIASEC dominan.

Mode Karir & Program Studi

Mode karir mengurutkan skor RIASEC, membentuk kode Holland tiga huruf teratas melalui utilitas badge, lalu menampilkan lencana profil bila tersedia. Halaman hasil juga memuat grafik bar horizontal, tiga kepribadian dominan, tabel profesi, dan rekomendasi program studi berdasarkan data careers serta personalities.

Pengumpulan Jawaban

Jawaban siswa disimpan sebagai baris individual pada ResultAnswer dengan field section, question, dan answer. Nilai jawaban default adalah "Selected". Pola ini membuat hasil tidak hanya berisi skor agregat, tetapi juga bisa ditelusuri sampai pertanyaan yang dipilih.

05 · Architecture

Arsitektur Sistem

Aplikasi adalah full-stack Next.js App Router dengan React client components untuk wizard dan dashboard interaktif, API route handlers untuk operasi server, Prisma Client untuk akses PostgreSQL, serta utilitas domain untuk scoring dan ekspor.

Stack Teknologi

LapisanTeknologiCatatan
FrameworkNext.js 16.2.2, App RouterServer components untuk halaman admin; route handlers untuk API.
UIReact 19.2.4, TypeScript, Tailwind CSS 4Client components untuk wizard, chart, form, dan ekspor.
DatabasePostgreSQLConnection string dari DATABASE_URL.
ORMPrisma 7.8.0 + @prisma/adapter-pgClient generated ke prisma/generated/client.
Authjsonwebtoken, bcryptjsJWT 24 jam, password bcrypt cost 12.
ChartChart.js, react-chartjs-2Dipakai pada visualisasi hasil.
ExportBrowser print, custom XLSX generatorExcel dibuat client-side dari XML dan ZIP minimal.

Struktur Direktori

src/app/admin              Halaman login, register, dashboard, detail sesi, detail hasil
src/app/api                API auth, admin users, sessions, collaborators, results
src/app/test/[code]        Entry point tes siswa berdasarkan kode sesi
src/components             Wizard, hasil peminatan/karir, navigasi, chart/table
src/data                   Pertanyaan, kepribadian, karir, badge, bobot peminatan
src/lib                    Auth, Prisma, generator kode, session access
src/utils                  Scoring peminatan dan utilitas PDF
prisma                     Schema, migration, seed admin

Boundary Server dan Client

  • Server membaca cookie auth dan mengambil data Prisma untuk halaman dashboard/detail.
  • Client component mengelola interaksi form, toggle, modal, wizard, chart, dan ekspor.
  • API admin memanggil requireAuth atau requireAdmin sebelum membaca atau mengubah data.
  • API siswa untuk session lookup dan result submission tidak memerlukan login, tetapi dibatasi oleh kode sesi, status aktif, dan mode sesi.

Keputusan Arsitektur

  • Prisma dipilih agar schema, migration, dan type generation berada dalam workflow yang sama.
  • JWT custom dipakai karena kebutuhan auth sederhana: email/password, role, status, dan cookie server-side.
  • Wizard tetap client-side agar siswa tidak menunggu request server di setiap langkah pertanyaan.
  • Jawaban pertanyaan dinormalisasi ke tabel terpisah agar analitik pertanyaan bisa ditambahkan di masa depan.
  • Session sharing dipisah lewat tabel junction SessionCollaborator agar owner dan akses bersama tetap jelas.

Catatan Konsistensi Spesifikasi

  • OpenSpec awal menyebut kode sesi 8 karakter base62, tetapi implementasi sekarang memakai slug dari nama sesi dengan suffix numerik bila terjadi bentrok.

  • Schema saat ini memakai model User yang dipetakan ke tabel users, bukan model Admin terpisah.
07 · Operations

Keamanan, Operasional, dan Deployment

Aplikasi membutuhkan Node.js, PostgreSQL, environment variable database dan JWT, migrasi Prisma yang sudah diterapkan, serta seed admin pertama sebelum dipakai oleh sekolah.

Environment Variable

NamaWajibFungsi
DATABASE_URLYaConnection string PostgreSQL untuk Prisma dan seed.
JWT_SECRETYaSecret penandatangan JWT; aplikasi akan error bila tidak tersedia saat auth dipakai.
ADMIN_EMAILTidakEmail admin awal saat seed; default admin@example.com.
ADMIN_PASSWORDTidakPassword admin awal saat seed; default admin123.

Perintah Lokal

npm install
npx prisma migrate deploy
npx prisma db seed
npm run dev
npm run build
npm run start
npm run lint

Checklist Deployment

  1. Siapkan PostgreSQL dan isi DATABASE_URL.
  2. Isi JWT_SECRET dengan nilai panjang dan acak.
  3. Jalankan npx prisma migrate deploy di environment target.
  4. Jalankan npx prisma db seed sekali untuk membuat admin awal bila belum ada.
  5. Build aplikasi dengan npm run build.
  6. Login sebagai admin, approve guru, buat sesi uji, buka link tes, submit hasil, dan cek dashboard.

Keamanan dan Kontrol Akses

  • Password tidak disimpan plaintext; hash dibuat dengan bcrypt.
  • Cookie JWT memakai httpOnly, sameSite lax, dan secure saat production.
  • requireAuth menolak token kosong, token invalid, akun PENDING, dan akun REJECTED.
  • requireAdmin membatasi endpoint pengelolaan guru dan kolaborator.
  • Akses detail sesi untuk guru dibatasi oleh ownership atau kolaborasi; admin memakai aturan khusus pada helper session access.
